Agente de IA oficial de Anthropic para la terminal. Lee tu código, ejecuta comandos, modifica archivos y conecta con servicios externos vía MCP. Ideal para desarrollo interactivo y automatización.

Instalación
curl -fsSL https://claude.ai/install.sh | bash

Requiere Node.js 18+ y API key de Anthropic.

Comandos Esenciales
Comando Descripción
claude Iniciar sesión interactiva
claude "prompt" Iniciar con prompt inicial
claude -p "prompt" Modo no-interactivo (sale tras responder)
claude -c Continuar sesión anterior
claude auth login Autenticarse con Anthropic
claude mcp Gestionar servidores MCP
Ejemplo Práctico
# Analizar un archivo de código
claude -p "What does this function do?" archivo.py

# Autenticarse para uso con API billing
claude auth login --console

# Continuar sesión anterior
claude -c
Documentación oficial de Claude Code
IA Responsable

Claude Code envía código a servidores de Anthropic. No expongas secretos, clavesAPI o datos sensibles en tus prompts. Revisa sempre el código generado antes de ejecutarlo. El modelo por defecto es Sonnet (~€3-€15por millón de tokens).

Asistente de IA de GitHub para tu terminal. Responde preguntas, escribe y depura código, interactúa con GitHub. Incluido con suscripción a GitHub Copilot.

Instalación
# macOS
brew install --cask github-copilot

# O con npm
npm install -g @github/copilot-cli
Comandos Esenciales
Comando Descripción
copilot Iniciar sesión interactiva
copilot -p "prompt" Modo no-interactivo
copilot --allow-tool Permitir herramientas
copilot /login Autenticarse
copilot /agent Invocar agente específico
copilot --resume Reanudar sesión
Ejemplo Práctico
# сделать pregunta directa
copilot -p "Explícame cómo funciona este archivo"

# Ejecutar comando de shell sin IA
!git clone https://github.com/owner/repo

# Solicitar revisión de código
copilot -p "Revisa los últimos cambios" --allow-tool='shell,read'
Documentación oficial de GitHub Copilot CLI
IA Responsable

Requiere suscripción a GitHub Copilot. Configura permisos de herramientas con --allow-tool para controlar qué puede hacer Copilot. Usa el argumento ! para ejecutar comandos directamente sin IA.

Agente de IA de código abierto. Flexible y extensible con soporte para múltiples proveedores de modelos (OpenAI, Anthropic, Ollama, etc.). Ideal para personalización.

Instalación
# Instalador oficial
curl -fsSL https://opencode.ai/install | bash

# O con npm
npm install -g opencode-ai

# O con Homebrew
brew install anomalyco/tap/opencode
Comandos Esenciales
Comando Descripción
opencode Iniciar TUI interactiva
opencode run "prompt" Ejecutar prompt directamente
opencode auth login Configurar API keys
opencode agent create Crear agente personalizado
opencode mcp add Agregar servidor MCP
opencode models Listar modelos disponibles
Ejemplo Práctico
# Iniciar sesión interactiva
opencode

# Autenticarse con proveedor
opencode auth login

# Ejecutar tarea específica
opencode run "Explain this code snippet"

# Crear agente personalizado
opencode agent create

IA Responsable

Utiliza API keys de Modelos.dev o proveedores propios. Puedes controlar qué herramientas puede usar cada agente. Muy útil para equipos que necesitan control total sobre sus datos.

Ejecuta modelos de lenguaje grandes localmente en tu máquina. Privacidad total ya que los datos nunca salen de tu equipo. Perfecto para investigación con datos sensibles.

Instalación
# macOS / Linux
curl -fsSL https://ollama.com/install.sh | sh

# Windows (WSL o Docker)
# Descarga desde ollama.com
Comandos Esenciales
Comando Descripción
ollama run modelo Ejecutar modelo (ej: llama3.2)
ollama pull modelo Descargar modelo
ollama list Listar modelos descargados
ollama serve Iniciar servidor API
ollama ps Ver modelos activos
ollama launch Iniciar integraciones (OpenCode, Claude)
Ejemplo Práctico
# Descargar y ejecutar modelo
ollama pull llama3.2
ollama run llama3.2

# Ver modelos disponibles
ollama list

# Iniciar servidor API local
ollama serve

# Lanzar integración con Claude Code
ollama launch claude --model qwen3.5

Guías MCPs

El Model Context Protocol (MCP) es un estándar abierto para conectar aplicaciones IA con sistemas externos. Aprende a configurar servidores MCP para expandir las capacidades de tu CLI favorita.

MCP (Model Context Protocol) es como el USB-C para aplicaciones de IA. Proporciona una forma estándar de conectar modelos de IA a fuentes de datos externas, herramientas y flujos de trabajo. Pensado como un "conector universal" entre la IA y el mundo exterior.

  • Recursos: Datos que la IA puede leer (archivos, bases de datos)
  • Herramientas: Acciones que la IA puede ejecutar (API calls, comandos)
  • Prompts: Plantillas reutilizables para workflows
Documentación oficial de MCP

Servidores MCP más utilizados por la comunidad:

MCP Server Función
GitHub Gestión de repos, issues, PRs
Filesystem Operaciones con archivos locales
Memory Memoria persistente (knowledge graph)
Git Operaciones Git
Sentry Monitoreo de errores
PostgreSQL Consultas a bases de datos
Context7 Búsqueda en documentación
Brave Search Búsqueda web
Repositorio oficial de servidores MCP

Agrega servidores MCP en tu archivo de configuración:

MCP Local
{ "mcp": { "mi-servidor": { "type": "local", "command": "npx", "args": ["-y", "@modelcontextprotocol/server-git"] } } }
MCP Remoto
{ "mcp": { "sentry": { "type": "remote", "url": "https://mcp.sentry.dev/mcp", "oauth": {} } } }
Gestión de MCPs
# Listar MCPs configurados
opencode mcp list

# Autenticar con MCP
opencode mcp auth mi-servidor

# Agregar MCP desde JSON
opencode mcp add-json nombre '{"type":"http","url":"..."}'
Documentación de MCPs en opencode

Gestiona servidores MCP directamente desde CLI:

# Agregar servidor MCP
claude mcp add nombre-servidor

# Listar MCPs
claude mcp list

# Ver detalles de un servidor
claude mcp get github

# Remover servidor
claude mcp remove github

# Agregar desde JSON
claude mcp add-json weather '{"type":"http","url":"..."}'
Configuración en archivo

También puedes configurar en ~/.claude/settings.json:

{ "mcpServers": { "github": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-github"] } } }

Documentación Skills

Las skills son módulos reutilizables que extienden las capacidades de tu asistente de IA. Cada CLI tiene su propio sistema de skills con configuraciones específicas.

Las skills en Copilot CLI son conjuntos de instrucciones que se cargan automáticamente para tareas específicas. Se definen en la configuración del proyecto.

Estructura de una Skill
.copilot/
├── commands/
��� └── summarize.md # /summarize command
└── skills/
└── reviewer.md # Skill completa con contexto
Ejemplo de Skill
--- name: code-reviewer
description: Realiza revisión de código
instruction: \|Eres un experto revisor de código. Analiza el código proporcionado
y sugiere mejoras. Enfócate en: seguridad, rendimiento y buenas prácticas.
tools: read,shell
Documentación de Skills en Copilot CLI

Las skills en Claude Code son módulos reutilizables con recursos incluidos (instrucciones, archivos de ejemplo, etc.). Se invocan con /nombre-skill dentro de la sesión.

Crear una Skill
# En sesión interactiva
/skill-create

# O manualmente en .claude/skills/
.claude/skills/
└── mi-skill/
├── skill.md
└── resources/
└── examples/
Estructura de skill.md
--- name: mi-skill
description: Análisis de código
tools: read,edit,glob
--- Eres un experto en análisis de código.
Ayúsame a identificar patrones y posibles errores.
Documentación de Skills en Claude Code

Los agentes en opencode son assistants especializados con prompts personalizados, modelos específicos y herramientas habilitadas.

Crear Agente
# Comando interactivo
opencode agent create

# O manualmente en agent.json
{ "agent": { "mi-agente": { "description": "Especialista en código Python", "model": "anthropic/claude-sonnet-4-20250514", "prompt": "{file./prompts/python-expert.txt}", "tools": { "read": true, "edit": true, "glob": true, "bash": true } } } }
Agentes Incorporados
  • build: Agente por defecto para desarrollo
  • plan: Análisis y planificación sin modificar código
Documentación de Agentes en opencode

principios para Skills Responsables
  • 1. Transparencia: Documenta qué hace cada skill
  • 2. Límites claros: Define qué NO puede hacer
  • 3. Validación: Incluye verificaciones de seguridad
  • 4. versionado: Controla cambios en el código
  • 5. Reutilización: Comparte skills con la comunidad
Estructura Recomendada
mi-proyecto/
├── .claude/ # Configuración de Claude Code
├── .copilot/ # Configuración de Copilot CLI
├── .opencode/ # Configuración de opencode
├── skills/ # Skills compartidas
└── agents/ # Agentes personalizados
